## 四、四问建模

### 4.1 问题一：时间对齐（无噪声）

**方法**：三次样条 + 粗搜(±50s, 0.1s步长) + 黄金分割精搜(精度 1e-6s)

**代码**：`code/models/problem_1_align.py`

**结果**：
- Δτ = **-198.43 s**
- RMSE = **8.39×10⁻¹¹ m**（浮点精度极限）
- 输出 10Hz 轨迹 7004 点

**验证**：`code/verifications/verify_1.py`（4/4 PASS）

### 4.2 问题二：联合估计 + KF 融合

**方法**：Nelder-Mead 联合最小二乘（3参数）+ 常加速度卡尔曼滤波（6维状态，异步更新）

**代码**：`code/models/problem_2_fuse.py`

**结果**：
- Δτ = **-48.44 s**
- Δx = **-3.44 m**，Δy = **1.81 m**
- RMSE = **1.56 m**
- 输出 8513 点

**验证**：`code/verifications/verify_2.py`（5/5 PASS）

### 4.3 问题三：偏差假设检验 + 融合

**方法**：联合估计 → Bootstrap(150次) 协方差 → Wald 检验 + AIC 对比双判据

**代码**：`code/models/problem_3_realdata.py`

**结果**：
- Wald W = **2.07**，p = **0.356** > 0.05
- ΔAIC = **-2.81** < 2
- **判决：H₀ 无显著系统偏差**
- 输出 3691 点（供 P4）

**验证**：`code/verifications/verify_3.py`（5/5 PASS）

### 4.4 问题四：任务规划

**方法**：SG 平滑(窗口151点=15s) → 差分得 v/a → 可行窗口扫描 → 贪心调度

**约束**（从 docx MathType OLE 提取）：
- 射击：d∈[5,30]m，v≤2m/s，a≤1.5m/s²，准备 1.5s
- 拍照：d∈[10,40]m，v≤1.5m/s，a≤1.5m/s²，准备 0.5s，角差≥60°

**代码**：`code/models/problem_4_schedule.py`

**结果**：
- 射击 **8** 次 + 拍照 **16** 次 = **24 个任务**
- 求解时间 1.2s

**验证**：`code/verifications/verify_4.py`（3/3 PASS）

## 九、TikZ 流程图设计规范

### 9.1 设计哲学

国一流程图的核心是**信息密度 + 视觉层次**，不是花哨。黑白打印下必须清晰可辨。

### 9.2 样式体系（6 层）

```latex
% preamble.tex 中定义
corebox   → 核心算法步骤（灰底+阴影浮起，最重视觉权重）
procbox   → 普通处理步骤（白底黑框，中等权重）
terminal  → 开始/结束（圆角，轻量）
decision  → 判断分支（菱形）
datanode  → 数据产物（梯形，区分于处理步骤）
badge     → 编号徽章（黑底白字圆形，附着在核心步骤左侧）
```

### 9.3 双列布局原则

```
左列 = 方法主干（自上而下，corebox 节点）
右列 = 数据产物（datanode，与左列一一对应）
连接 = 三种箭头语义：
  实粗线 arrow      → 主流程顺序
  点虚线 dataarrow  → 数据产出/传递
  粗虚线 feedback   → 验证失败回路
```

### 9.4 阴影效果

```latex
blur shadow={shadow blur steps=5, shadow xshift=0.5mm, shadow yshift=-0.5mm}
```
仅用在 corebox 和 terminal 上（2 种），不要全部加阴影（视觉噪音）。

### 9.5 分组框

```latex
groupbg/.style = {draw=black!40, dashed, thick, fill=black!3,
                  rounded corners=5pt, inner sep=8pt}
```
用 `\begin{scope}[on background layer]` 画在底层。标签用 `label={above left:...}` 避免被节点遮盖。

### 9.6 四问原理图设计模板

| 问题 | 图类型 | 核心元素 |
|---|---|---|
| P1 | XY 曲线示意 | 两条曲线 + Δτ 箭头 + 残差标注 + 目标函数文字 |
| P2 | 水平 pipeline | 输入梯形 → 处理方框 → 输出梯形，箭头连接 |
| P3 | 判定树 | 菱形判断节点 + 是/否分支 + 圆角结果框 |
| P4 | 时间轴 | 横轴时间 + 速度曲线 + 可行窗口填充 + 任务标记点 |

### 9.7 避坑清单

- 节点间距用 `node distance=0.8cm and 1.5cm`（过密看不清，过疏浪费版面）
- 中文字体用 `\small` 或 `\footnotesize`（正常大小在 tikz 里太大）
- 箭头统一用 `-Stealth`（不要混用 `->` 和 `->`）
- fit 节点必须在 `on background layer` 里画（否则遮盖内容）

## 十、摘要写作规范

### 10.1 页面预算

```
A4 纸面：297mm - 上下边距 50mm = 247mm 可用
行距：小四(12pt) × 1.38 = 16.56pt ≈ 5.84mm/行
可用行数：247mm / 5.84mm ≈ 42 行

分配：
  标题区（二号黑体+间距）    → 5 行
  "摘 要" 标题+装饰线       → 3 行
  正文区                    → 32 行
  关键词                    → 2 行
  
32 行 × ~24 字/行（含公式占宽）≈ 760-820 字
```

### 10.2 段落结构（黄金比例）

```
开头段（背景+概述）     60 字   ← 最短，一笔带过
问题一段              130 字   ← 方法+核心公式+结果数字
问题二段              150 字   ← 最复杂的问题给最多篇幅
问题三段              140 字   ← 检验框架+判决结果
问题四段              150 字   ← 约束量化+调度结果
创新点段              100 字   ← 3-4 条，每条一句话
关键词                 30 字
合计                 ~760 字
```

### 10.3 每段必含要素

```
\textbf{针对问题X，} + 方法名(1句) + 核心模型/公式(1句) + 求解策略(1句) + 关键数值结果(1句，加粗) + 输出规格(半句)
```

### 10.4 加粗规则

| 加粗 | 不加粗 |
|---|---|
| 最终结论数值（Δτ、RMSE、任务数） | 中间过程数值（搜索范围、步长） |
| 判决结论（"无显著系统偏差"） | 方法名称 |
| 输出频率/点数（10Hz、7004点） | 公式符号定义 |

格式：`\textbf{$\Delta\tau = -198.43$\,s}`（数值+单位一起加粗）

### 10.5 标点规范

- 全角中文标点：，。；：（）
- 数学公式内用英文逗号/句号
- 分号 "；" 用于并列分句（不是逗号）
- 公式后紧跟中文用全角逗号

### 10.6 关键词选择

5 个词，覆盖：核心方法(2) + 核心问题(2) + 特色技术(1)
```
多源融合定位（问题域）；联合参数估计（方法特色）；卡尔曼滤波（核心工具）；Wald 检验（问题三特色）；贪心优化（问题四方法）
```

## 十一、去 AI 腔写作指南

### 11.1 禁用词表（正文中绝不出现）

| 类别 | 禁用词 | 替代 |
|---|---|---|
| 工程实现 | 脚本、代码、Python、函数、调用 | 不提，或放附录 |
| 过度承诺 | 确保、保证、完备、完美 | 验证、支持、满足 |
| 营销感 | 全流程、配套、开箱即用、一键 | 删除 |
| AI 特征 | 100%、逐坐标、逐约束核验通过 | "全部满足"、"逐一验证" |
| 比较夸张 | 远优于、显著优于、大幅提升 | "优于"、"低于"、"改善" |
| 虚话 | 具有理论保证、具备良好的 | 给具体定量（"近似比≤2"） |

### 11.2 人类写作风格特征

**AI 写的**：
> 本文采用的贪心启发式策略具有理论保证，相关方法参考组合优化经典文献[6]。

**人写的**：
> 贪心策略的近似比分析参见文献[6]。

**AI 写的**：
> 全流程可复现：固定随机种子、数据路径、Python 依赖版本。

**人写的**：
> （直接删除——国赛论文从不写这些）

**AI 写的**：
> 问题四自证体系完备：独立验证脚本对每行 result.xlsx 逐约束核验，可确保解的合法性。

**人写的**：
> 问题四求解结果经逐任务约束满足性验证，所有任务的距离、速度、加速度及时序约束均严格满足。

### 11.3 改写核心原则

1. **只谈数学，不谈工程**：正文是数学论文，不是技术博客
2. **结论式表述**：直接说"满足约束"，不说"通过验证确保满足约束"
3. **定量取代定性**："优于 DTW 约一个数量级" 而非 "远优于 DTW"
4. **删除冗余修饰**：一句话能说清的不用两句
5. **被动/第三人称**："本文建立了…" 而非 "我们建立了…"

### 11.4 模型评价章写法

**错误**（AI bullet list）：
```
\begin{itemize}
\item 优点一：问题一采用闭式解，复杂度 O(N log N)，远优于 DTW
\item 优点二：联合估计避免误差累积
\item 全流程可复现：固定随机种子
\end{itemize}
```

**正确**（自然段落）：
```
本文模型在数学性质上有如下特点。问题一的样条对齐方法代价函数
严格凸，保证全局最优唯一，计算复杂度 O(N log N)，低于 DTW 的
O(N²)。问题二的联合估计将三参数纳入同一框架，避免分步法误差
传播。[...]

模型的主要局限在于：（1）贪心策略非严格最优......
```

```latex
\setlength{\belowcaptionskip}{2pt}                   % 题注与正文
```

### 12.5 章节标题层次感

```
section    → 小二黑体 + 底部 0.8pt 横线（最重）
subsection → 三号黑体（中等）
subsubsection → 四号黑体（最轻）
```

间距递减：section 上 18pt / subsection 上 12pt / subsubsection 上 8pt

### 12.6 高级感突出技巧

1. **装饰线分隔**：标题下、摘要标题下各一条 `\rule{0.5\linewidth}{0.6pt}`
2. **badge 编号**：流程图用黑底白字圆形 ① ② ③ ④ 替代纯文字编号
3. **阴影浮起**：核心节点加 blur shadow，其余不加（对比产生层次）
4. **灰底分组框**：`fill=black!3` 的虚线圆角框包裹核心建模区
5. **三种箭头**：实线=主流、虚点=数据流、粗虚=反馈（语义分层）
6. **符号表扩充**：24 行 × 3 列满版，行距 1.6（不留空白）
7. **代码框**：浅灰底 + 左侧行号 + 单线边框（listings 包）

## 十三、关键经验教训

1. **数据先行**：所有建模前必须 EDA 确认数据特征（附件3 时段不重叠是决定性发现）
2. **自证必须**：每个模型配验证脚本，先跑通 verify 再写论文
3. **SG 平滑是 P4 命脉**：噪声轨迹直接差分 v/a 全部爆炸，必须平滑
4. **全文数据一致**：P4 经过 SG 窗口从 101→151 的升级，所有引用处必须同步
5. **去 AI 腔**：正文不提"脚本/代码/Python"，只谈数学验证手段
6. **阈值提取**：docx 中 MathType OLE 公式无法程序提取，必须人工读 WMF 图
7. **浮动体策略**：算法环境 70 行用 [H] 必爆空白页，改 [htbp] + 全局参数调优
8. **摘要精控**：A4 小四 1.38 倍行距一页约 760-820 字，超了就溢出
9. **Unicode 陷阱**：①②③④ 在主字体缺字形时显示方块，用（1）（2）（3）（4）最稳
10. **表格填版**：符号表/灵敏度表行数不够半页时，扩充行数 + 加大 arraystretch
11. **附录实填**：空附录比没有附录更减分，至少贴 20-30 行核心代码 + 数学推导
12. **迭代审校**：每次大改后重新编译检查页数、空白、乱码（至少 12 轮迭代）
